    • There are several menu's available from the menu system, specific for the master grid.
    • The content type "master page" can be used for the Master grid content. Content for this content type is translatable.
    • The "master page" content does not necessarily need to be created first, but it MUST exist in English to have a correct menu and breadcrumbs.
    • The "variations" for different profiles is NOT included. If must be build, we would use "minipanels" or some other custom API to achieve the profile-driven blocks.
